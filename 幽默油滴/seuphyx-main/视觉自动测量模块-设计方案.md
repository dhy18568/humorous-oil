## 1. 背景知识：OpenCV 是什么、为什么用它

### 一句话解释

**OpenCV**（Open Computer Vision，开源计算机视觉库）是一个给电脑"看东西"的工具箱。就像你手机上的美颜相机知道你的脸在哪、门禁系统能识别车牌号——它们背后用的就是类似 OpenCV 这样的技术。

### 它和我们这个项目有什么关系

你的显微镜摄像头通过 USB 采集卡接入电脑后，OpenCV 可以：

| 功能 | 对应我们项目里的用途 |
|------|-------------------|
| 从采集卡逐帧读取画面 | 代替 PotPlayer，程序内部直接看到画面 |
| 在画面上找特定的东西 | 找到网格线、找到发光的油滴 |
| 逐帧跟踪一个移动物体 | 跟着一颗油滴，看它每一帧移动到了哪里 |
| 识别画面上的数字 | 读取右上角的电压值 |

### 它怎么安装

```bash
pip install opencv-python
```

然后 Python 里 `import cv2` 就能用。它是纯软件，不需要额外硬件，不修改任何系统设置。和 PotPlayer 一样，只是"看"采集卡画面，不冲突。

## 2. 硬件原理：摄像头如何被程序读取

### 你的设备和电脑之间的连接

```
显微镜镜头
    │
    ▼
USB 视频采集卡 ──USB线──▶ 电脑
    │                       │
    │                 ┌─────┴──────┐
    │                 ▼            ▼
    │             PotPlayer    我们的程序
    │           （人工观看）  （自动读取+处理）
    │
    │   两者看到的是同一个画面源
    │   就像两个软件可以同时打开同一个摄像头
```

### USB 采集卡在电脑里是什么

Windows 把它识别为一个**标准摄像头设备**。这跟笔记本自带的摄像头、插在显示器上的 USB 摄像头没有本质区别——对电脑来说，它们都是"一个能出画面的设备"。

OpenCV 里打开它只需要一行代码：

```python
cap = cv2.VideoCapture(0)  # 0 = 第一个摄像头设备
```

如果 `0` 是你笔记本自带的摄像头，改成 `1` 或 `2` 就能切到采集卡。

### 关于"保存视频"

**整个过程不会保存任何视频文件到硬盘**。程序的处理方式是：

```
摄像头送来第 1 帧 → 显示在网页上 → 检查油滴位置 → 丢弃
摄像头送来第 2 帧 → 显示在网页上 → 检查油滴位置 → 丢弃
摄像头送来第 3 帧 → 显示在网页上 → 检查油滴位置 → 丢弃
...
```

就像你眼睛看到东西、脑子处理、然后忘了——从来不需要"录下来"。所有数据都在内存里，关了程序就没了，不会占用硬盘空间。

## 4. 阶段 1：记录平衡电压

### 做什么

此时显微镜上电压开着，油滴悬浮静止。画面右上角会显示电压数值（比如 `187 V`）。这一步就是把这个电压值记下来。

### 怎么交互

```
┌──────────────────────────────────────────────┐
│             实时摄像头画面                     │
│                                    ┌───────┐ │
│                                    │ 187 V │ │ ← 显微镜自带叠加的电压数字
│                                    └───────┘ │
│                                              │
│     网格线可见，油滴悬浮                       │
│                                              │
└──────────────────────────────────────────────┘
  [📸 记录平衡电压]  ← 点击这个按钮
```

点击按钮后，程序抓取当前帧，截取右上角一小块区域：

```
┌──────────────────────────────────────────────┐
│  程序截取的区域（约 100×40 像素）：             │
│  ┌──────────────────────┐                    │
│  │                 187 V│   ← 只有这块被送给  │
│  └──────────────────────┘      OCR 引擎识别   │
└──────────────────────────────────────────────┘
```

### OCR 识别电压

程序用 **EasyOCR**（一种离线可用的文字识别引擎）读取截取区域的数字。

然后弹出一个确认框：

```
┌─────────────────────────────────────┐
│  检测到平衡电压：187 V               │
│                                     │
│  [✅ 确认，使用此值]                 │
│  [✏️ 手动修改]  [___] V             │
└─────────────────────────────────────┘
```

### 状态更新

确认后，电压值存入 `st.session_state.balance_voltage`，状态栏更新：

```
🟢 已完成  ✅ 平衡电压：187V
🔵 当前步骤：请在实时画面中点击网格线设定起点线
⚪ 待完成：设定终点线、选油滴、测量、录入
```

### 技术实现要点

- **OCR 引擎**：使用 `easyocr.Reader(['en'])`，初始化一次
- **截取区域**：用户可选"电压数字在画面的哪个区域"（提供"右上角""左上角""右下角""左下角"四个选项，默认右上角），截取 120×50 像素
- **预处理**：截取后转灰度 → 放大 2 倍（提高 OCR 识别率）→ 自适应二值化 → 送给 OCR
- **数字提取**：OCR 结果中取第一个匹配数字的值
- **容错**：如果 OCR 返回空，提示用户手动输入，不阻塞流程

## 5. 阶段 2：设定起点线和终点线

### 做什么

让学生在实时画面中指定两条网格线作为"测量标记线"。油滴从上往下落，越过上面那条（起点线）开始计时，越过下面那条（终点线）停止计时。

### 为什么能用网格线当标尺

显微镜的视野里永远有等间距的网格线（间距 0.25mm）。这些网格线是**内置在光学系统中的标尺**——无论摄像头怎么摆、怎么歪，网格线都在那里，间距不变。

### 标定原理（防抖动）

每次启动时，学生点两次：

```
学生点击"第 3 条网格线" → 系统记录像素位置 y₃
学生点击"第 8 条网格线" → 系统记录像素位置 y₈

格数差 = 8 - 3 = 5 格
物理距离 = 5 × 0.25mm = 1.25mm（不变！）
像素间距 = |y₈ - y₃| px

当前标定：px_per_mm = 像素间距 / 1.25
```

之后在实时跟踪阶段，**每一帧都会重新检测网格线的实际像素位置**（因为摄像头可能有微小抖动），然后用最新检测到的位置微调起点线和终点线在画面上的 y 坐标。这样即使摄像头晃动，两条线始终咬在正确的物理位置。

### 交互流程

**第一步**——提示：

```
🔵 当前步骤：请点击画面中的一条网格线作为【起点线】（油滴先经过的那条，靠上）
💡 提示：起点线和终点线应跨越多条网格线（建议 5~8 格），以确保测量的下落距离足够
```

实时画面叠加一个可点击的画布。用户鼠标点击一条网格线后：

```
🟢 已完成  ✅ 起点线已设定（绿色）
🔵 当前步骤：请点击画面中的另一条网格线作为【终点线】（靠下的那条）
```

用户点击第二条：

```
🟢 已完成  ✅ 起点线  ✅ 终点线
   ┌─────────────────────────────────────┐
   │  标定信息：                          │
   │  起点线：画面 y = 218 px             │
   │  终点线：画面 y = 598 px             │
   │  格数差：5 格                        │
   │  物理距离：1.25 mm（固定）            │
   │  像素比例：304 px/mm                 │
   └─────────────────────────────────────┘
🔵 当前步骤：请捕获当前帧并点击油滴
```

### 画面叠加效果

在实时画面中：

```
┌──────────────────────────────────────────────┐
│                                              │
│    ██████████ 起点线（绿色）██████████        │  ← 始终叠加在所有帧上
│    ───────── 网格线 ──────────────            │
│    ───────── 网格线 ──────────────            │
│    ───────── 网格线 ──────────────            │
│    ───────── 网格线 ──────────────            │
│    ██████████ 终点线（红色）██████████        │  ← 始终叠加在所有帧上
│    ───────── 网格线 ──────────────            │
│                                              │
│    ●   ← 悬浮中的油滴（还没开始测量）          │
│                                              │
└──────────────────────────────────────────────┘
```

### 技术实现要点

- **画布组件**：使用 `streamlit-drawable-canvas` 或自定义 JS 组件，在实时帧画面上叠加透明画布，捕获鼠标点击坐标
- **网格线吸附**：点击后，在点击位置 ±30 px 范围内搜索实际网格线位置（用 Canny 边缘检测 + 水平线检测），自动将线吸附到最近的水平网格线上
- **每帧微调**：实时跟踪阶段，对每帧做相同的网格线检测，用新检测到的位置更新线的 y 坐标，实现抗抖动
- **颜色规则**：
  - 绿色（半透明）：起点线，油滴未触发
  - 红色（半透明）：终点线，油滴未触发
  - 黄色（闪烁）：已触发（油滴越过了）
- **物理距离计算**：`distance_mm = 格数差 × 0.25`，存入 `st.session_state.measurement_distance_mm`

## 6. 阶段 3：选定油滴

### 做什么

冻结当前帧，让用户用鼠标在大致位置点一下油滴，程序自动找到油滴的精确中心。

### 为什么冻结帧而不是实时的

因为油滴此时还悬浮着（电压还开着、或者还没开始测量），它不动。冻结一帧让人安心点选，不会因为鼠标移动导致画面变化。这个冻结不影响时间测量——时间测量发生在阶段 4 才开始的跟踪中。

### 交互流程

**第一步**——用户点击 `[📸 捕获当前帧]` 按钮：

- 当前实时画面冻结，存为一张静态图
- 静态图显示在画布组件上（和实时画面一样大）
- 提示更新：

```
🔵 当前步骤：请用鼠标点击画面中要测量的油滴（大致位置即可）
💡 提示：程序会自动在您点击的位置附近搜索油滴的精确中心
```

**第二步**——用户在画面上点击油滴的大概位置：

```

│          冻结的静态画面                        │
│                                              │
│   ██████████ 起点线（绿色）██████████         │
│                                              │
│         ┌─────────┐                          │
│    ·····│✦········│····  ← 点击位置附近       │
│    ·····│·  ●  ··│····    40×40 px 搜索窗    │
│    ·····│·········│····                      │
│         └─────────┘                          │
│           ↑ 放大预览（见下方）                  │
│                                              │
│   ██████████ 终点线（红色）██████████         │
│                                              │
└──────────────────────────────────────────────┘

┌────────────── 放大预览 ──────────────┐
│                                      │
│           ·············              │
│           ·····✦·····  ← 十字       │
│           ····●●●····    标出        │
│           ····●●●····    检测到的    │
│           ····●●●····    油滴中心    │
│           ·············              │
│                                      │
│  像素坐标：(342, 218)                 │
│  亮度峰值：247（背景平均 180）        │

└──────────────────────────────────────┘

  [✅ 确认此油滴]  [🔄 重新选择]
```

### 自动修正原理（对用户透明）

用户不需要精确点到油滴中心。程序在用户点击的坐标周围取一个 40×40 像素的小窗口，然后：

1. 计算这个小窗口里每个像素的灰度值
2. 找到最亮的一片区域（油滴比背景亮）
3. 判断这片区域是否接近圆形
4. 如果是，取圆心作为油滴精确定位中心
5. 如果不是（比如用户点到了网格线上），提示"未检测到油滴，请重新选择"

### 技术实现要点

- **搜索窗口**：以点击坐标为中心，40×40 px
- **检测方法**：
  1. 窗口内转灰度 → 高斯模糊（减少噪点）
  2. 自适应阈值二值化 → 找出亮斑
  3. 查找轮廓（`cv2.findContours`）→ 筛选面积 5~200 px² 的轮廓
  4. 用 `cv2.minEnclosingCircle` 拟合圆 → 选出最圆的亮斑
  5. 返回圆心坐标
- **放大预览**：用 PIL 或 OpenCV 将 40×40 窗口放大到 160×160（4 倍），叠加十字准星
- **油滴特征存储**：圆心坐标 + 窗口的灰度直方图（用于后续跟踪的外观参考）
- **失败处理**：没检测到 → 提示"此区域未检测到油滴，请尽量点击油滴中心位置后重试"，允许重试

## 7. 阶段 4：实时跟踪测量下落时间

### 这是最关键的一步

学生关掉电压后，油滴开始下落。程序需要在每一帧中跟踪油滴的位置，然后精确记录它越过起点线和终点线的时刻。

### 跟踪原理

程序使用**两种方法结合**来跟踪油滴：

#### 方法 A：帧差法（适合背景静止、物体移动的场景）

```
第 N 帧画面  ──相减──▶  第 N+1 帧画面
                        │
                        ▼
                  变动的部分 = 油滴移动的区域
                  （网格线、背景都不动，只有油滴在动）
```

因为显微镜视野里除了油滴之外所有东西（网格线、背景）都是静止的，相邻两帧相减，结果就是"油滴从哪移到了哪"。

#### 方法 B：光流法（跟踪角落/边缘特征点）

如果油滴有可识别的纹理（不是纯白圆点），光流法可以追踪它表面的一些特征点，知道它们从上一帧移到了下一帧的哪里。

#### 两种方法互补使用

- 帧差法给一个大致的移动区域
- 光流法在上述区域内精确定位油滴中心
- 如果某一帧光流法失败（比如油滴暂时模糊），就用帧差法的粗定位顶上

### 时间测量逻辑

这里的"时间"不是系统时钟，而是**帧号 ÷ 帧率**：

```python
# 伪代码（不是实际代码，只是说明逻辑）
帧率 = 30  # fps，从摄像头获取

第 1 帧：油滴 y=180  ← 还在起点线（y=200）上方
第 2 帧：油滴 y=185
...
第 45 帧：油滴 y=199 ← 还在起点线上方
第 46 帧：油滴 y=202 ← 首次越过起点线（y=200）！
          → t_start = 46 / 30 = 1.533 秒
...
第 145 帧：油滴 y=398 ← 还在终点线上方
第 146 帧：油滴 y=403 ← 首次越过终点线（y=400）！
          → t_end = 146 / 30 = 4.867 秒

下落时间 = 4.867 - 1.533 = 3.334 秒
```

**误差分析**：帧率 30fps 意味着每帧间隔 0.033 秒。所以测量误差上限是 ±0.033 秒（±1 帧），对于密立根实验来说绰绰有余。

### 异常情况处理

| 情况 | 处理 |
|------|------|
| 油滴跟踪丢失（移出画面等） | 提示"跟踪丢失，请检查油滴是否还在视野中，重试" |
| 油滴越过了终点线但没触发起点线 | 不可能（必须先过起点才到终点），但如果发生了，忽略 |
| 30 秒还没触发终点线 | 超时提示"油滴下落时间过长，请检查并重试" |
| 油滴始终没触发起点线（用户没关电压） | 60 秒超时提示"未检测到油滴移动，请确认已关掉电压" |

### 技术实现要点

- **帧差法实现**：
  ```python
  prev_gray = cv2.cvtColor(prev_frame, cv2.COLOR_BGR2GRAY)
  curr_gray = cv2.cvtColor(curr_frame, cv2.COLOR_BGR2GRAY)
  diff = cv2.absdiff(curr_gray, prev_gray)
  _, thresh = cv2.threshold(diff, 25, 255, cv2.THRESH_BINARY)
  ```
- **光流法实现**：用 `cv2.calcOpticalFlowPyrLK`，以油滴选中位置的特征点作为初始跟踪点
- **跟踪框大小**：以检测到的油滴圆心为中心，20×20 px（油滴很小的场景）
- **帧率获取**：`cap.get(cv2.CAP_PROP_FPS)`，如果返回 0（采集卡不支持），默认用 30
- **网格线实时微调**：每 5 帧做一次自动网格线检测，用新位置更新起点/终点线的 y 坐标
- **画面刷新**：用 `st.empty()` 创建占位符，循环写入新帧（PNG 格式），保持实时显示
- **计时精度**：`帧号差 / 实际帧率`，在 tracking 循环中用 `frame_index` 计数器

#### 帧率获取的详细说明

**不需要用户手动输入。** 程序会自动处理，分三层策略：

```
第 1 层：自动读取
         cap.get(cv2.CAP_PROP_FPS)  ← 从采集卡读取帧率
              ↓
         读到了靠谱的值（如 30、60）→ 直接用，用户无感
              ↓
         读到 0 或异常值（某些廉价采集卡报不准）
              ↓
第 2 层：使用默认值 30 fps
         这是绝大多数 USB 采集卡的真实帧率，准确度足够
              ↓
第 3 层：高级设置（手动覆盖）
         阶段 2 的设置面板里有一个折叠的"高级选项"
         → 展开后可以看到"摄像头帧率"输入框
         → 预填值为自动读取的结果
         → 用户只有在发现测量时间明显不准时才需要改
```

**为什么默认 30 就够用了？**
- 市面上 95% 的 USB 视频采集卡工作在 30fps
- 即使实际是 25fps、误差 5fps，对 3 秒下落时间的测量影响也只有约 0.6 秒——而且这种差距在第一次使用时就能发现并纠正
- 对于密立根实验，±0.033 秒的帧间误差远小于人为掐秒表的误差（约 ±0.2 秒）

**怎么判断帧率设置对不对？**
- 找一个已知时间（比如手机秒表同时计时），对比程序的测量结果
- 如果差异 > 0.1 秒，在高级设置里微调帧率值
- 通常一次校正后就不需要再调了

## 9. 关键技术细节

### 9.1 新增依赖清单

```txt

streamlit-drawable-canvas>=0.9.0  # 画布上的鼠标点击交互

```

添加到 `pyproject.toml` 的 `dependencies` 列表中。

### 9.2 摄像头与浏览器之间的数据流

```
USB 采集卡
    │
    ▼
cv2.VideoCapture(0)          ← Python/OpenCV 后端
    │
    ├── 实时模式：#1 阶段：每 0.1 秒取一帧
    │              → cv2.imencode('.jpg', frame)
    │              → st.image(jpg_bytes)
    │              → 在 Streamlit 页面上显示
    │
    ├── 冻结模式：#1 阶段 3：取当前帧
    │              → 存为静态图
    │              → 传给画布组件供点击
    │
    └── 跟踪模式：#1 阶段 4：每帧处理
                   → 帧差法 + 光流法 → 更新油滴位置
                   → 叠加起点线/终点线/十字准星
                   → 检测是否越线 → 记录时间
                   → 编码为 jpg → st.image() 显示
```

### 9.3 为什么帧差法适合这个场景

密立根油滴实验中：
- 摄像机的视野是**固定不动的**
- 网格线、背景、光路都是**不变**的
- 唯一的变化是**油滴在移动**

因此两帧相减，结果几乎就是"油滴的轨迹"。这是帧差法最理想的应用场景，计算量极小，实时性好。

### 9.4 网格线自动检测算法

```
输入：一帧灰度画面
步骤：
  1. Canny 边缘检测（低阈值 50，高阈值 150）
  2. 霍夫线检测（只检测接近水平方向，角度 ±5° 以内）
  3. 按 y 坐标对所有检测到的线排序
  4. 过滤：只保留间距均匀的线组（间距标准差 < 5 px 的线群 = 网格线）
  5. 返回检测到的网格线 y 坐标列表
输出：grid_lines = [y₁, y₂, y₃, y₄, y₅, ...]
```

然后根据用户阶段 2 选中的两条网格线在当前检测结果中的索引，更新起点/终点线的 y 坐标。

### 9.5 油滴亮度检测

油滴比背景亮（用户确认），因此搜索窗口内的逻辑是：

```python
# 伪代码
window = gray_image[click_y-20:click_y+20, click_x-20:click_x+20]
# 二值化：高于阈值 = 亮斑（油滴），低于阈值 = 背景
_, binary = cv2.threshold(window, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)
# 如果油滴比背景亮，OTSU 阈值会把亮的部分分离出来
# 查找亮斑的轮廓
contours, _ = cv2.findContours(binary, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
# 筛选最圆的亮斑
best = max(contours, key=lambda c: circularity(c))
center = centroid(best)
```

## 11. 非技术人员常见疑问

### Q1：这个程序会保存我的视频吗？
**不会。** 程序只在内存里"看一眼"每一帧画面，处理完就丢弃，不写入任何视频文件到硬盘。

### Q2：我的摄像头画面会不会被传到网上？
**不会。** 整个处理都在你的电脑本地完成，不联网、不上传。OCR 引擎（EasyOCR）也是离线工作的。

### Q3：PotPlayer 和这个程序能同时打开吗？
**可以。** 两个软件都能同时读取 USB 采集卡的画面。但建议测量时关掉 PotPlayer，避免两个软件抢占采集卡资源导致卡顿。

### Q4：摄像头歪了一点，网格线位置变了，要重新标定吗？
**不需要。** 程序在测量过程中会**自动检测每一帧的网格线位置**，实时微调起点线和终点线的 y 坐标。只要网格线还在画面里，就不怕微小的位移。

### Q5：油滴太小，我怕点不准。
程序会在你点的位置附近**自动搜索最亮的圆斑**（油滴比背景亮），并把搜索区域放大 4 倍让你确认。你只需要点到油滴的大概位置就行。

### Q6：怎么知道程序有没有在正常工作？
页面顶部的**状态栏**会随时告诉你当前的进度：
- 绿色 = 已完成
- 蓝色 = 当前正在进行
- 白色 = 待完成
每步都有明确的提示文字告诉你该做什么。

### Q7：摄像头的帧率需要我自己输入吗？
**不需要。** 程序会自动从采集卡读取帧率。读不到的话，默认使用 30fps（这是市面上绝大多数 USB 采集卡的真实帧率）。只有在极少数情况下（比如你的采集卡确实不是 30fps、测出来的时间明显不对），才需要去阶段 2 的高级设置里手动调整。详见上面"第 4 节"的帧率获取说明。

### Q8：测量时间准确吗？
测量精度取决于摄像头帧率。30fps 的摄像头误差约 ±0.033 秒，对密立根实验完全够用（远小于人手掐秒表的约 ±0.2 秒误差）。如果觉得不够，可以使用更高帧率的采集卡。

### Q9：同一个电压下要测多个油滴，电压要重新录吗？
**不需要。** 录一次电压后，测完一颗油滴会自动循环回"选油滴"步骤，电压保持不变，连续测就行。

### Q10：如果程序出错了怎么办？
任何错误都会在页面上显示红色提示，并说明原因。不会导致程序崩溃。按提示重试即可。
